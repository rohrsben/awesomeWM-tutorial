Default libraries

The functionality we want to add to awesome (like, you know, viewing windows) can be provided by awesome's default libraries. In this page, we will make our first edits to rc.lua.test by adding the rest of the defaults back in, with a brief explanation for each one. Everything is gonna get a more detailed explanation later on, so don't worry if some things don't make sense. Let's start with the ones that are already in there.

Gears

Gears pretty much provides methods for interacting with your computer itself. E.g. Cairo (the graphics library used by awesome), strings and timers, the filesystem, etc. The latter is why it's included in rc.lua.test, in fact; if you look at the first non-require line, you'll see gears.filesystem.get_xdg_config_home(). This is how awesome accesses the tutorial-background.png to set the background.

Awful

Awful is the library provided for interacting with application windows and hardware like screens, or the mouse and keyboard. Thus, the second line in rc.lua.default: awful.screen.connect_for_each_screen is what sets the background that gears accessed. connect_for_each_screen can do a lot more, but we'll get into that later.

Beautiful

Beautiful is awesome's theme library. It deals with how windows are styled, what fonts are used in titlebars, etc. Your theme is specified in a different file: currently, tutorial-theme/tutorial-theme.lua. The name can be anything you like, which also allows you to have multiple themes and switch between them. This theme file is what beautiful mostly interacts with/returns values from.



Since now is as good a time as any, let's talk vocab. In general, I'm going to try and use the words that awesome and X11 use for things. Since this vocab felt confusing to me at first, I'll try and mention when it's different from common parlance. With that in mind, what are commonly called "windows" will be referred to as "clients" from now on. Here's a test sentence for you to say: I'm reading this sentence in a client. Nice!
