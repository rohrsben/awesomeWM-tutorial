Default libraries

The functionality we want to add to awesome (like, you know, viewing windows) can be provided by awesome's default libraries. In this page, we will make our first edits to rc.lua.test by adding the rest of the defaults back in, with a brief explanation for each one. Everything is gonna get a more detailed explanation later on, so don't worry if some things don't make sense. Let's start with the ones that are already in there.

Gears

Gears pretty much provides methods for interacting with your computer itself. E.g. Cairo (the graphics library used by awesome), strings and timers, the filesystem, etc. The latter is why it's included in rc.lua.test, in fact; if you look at the first non-require line, you'll see gears.filesystem.get_xdg_config_home(). This is how awesome accesses the tutorial-background.png to set the background.
